# BizGuard v1.2.2 - Mejoras de Estabilidad y Auth Multi-Dominio

Fecha: 2026-03-21
Estado: Desplegado en `dist-bizguard/`

## Resumen de Cambios

### 1. Soporte Multi-Dominio Dinámico (Auth.js)
Se ha refactorizado la autenticación para que sea **100% dinámica**. Ya no depende de un dominio hardcodeado (`.nathium.com.ar`).
- **Detección de Host**: El sistema analiza el header `Host` de cada petición.
- **Cálculo de Dominio**: Extrae automáticamente el dominio base (ej. `.nathium.com.ar`, `.empresa.com`).
- **Cookies Compartidas**: PKCE, sesión y callback ahora se comparten correctamente entre subdominios del mismo cliente.
- **Callback Flexible**: Soporta redirecciones automáticas al host que inició la sesión (ej. `core2.nathium.com.ar:8081`).

### 2. Heartbeat Shield (Cloudflare 524 Fix)
Para reportes que demoran más de 100 segundos, se ha implementado un escudo de conectividad:
- **Keep-Alive**: Después de 55s sin respuesta, el proxy envía headers HTML y espacios cada 15s para evitar el timeout de Cloudflare.
- **Descarga vía Blob**: Si el reporte es un archivo (Excel/PDF), se sirve mediante un script JS que inicia la descarga en el navegador una vez que los datos llegan completos.

### 3. BizGuard Performance UI
Se agregaron métricas de tiempo real en el dashboard:
- **Card HB Shield**: Indica cuántos procesos están bajo mantenimiento de conexión activa.
- **Fix Métricas Windows**: Cálculo de CPU corregido para entornos Windows (antes mostraba 0%).
- **Ping Robusto**: Los conectores ahora muestran "Online" incluso si devuelven errores 500 o tienen certificados SSL inválidos (Ignore SSL activado para health checks).

## Archivos Modificados
- `src/auth.ts`: Refactorización a inicialización asíncrona dinámica.
- `src/lib/proxy-server.ts`: Implementación de Heartbeat Shield y Blob Download.
- `src/lib/proxy-manager.ts`: Gestión de `heartbeatCount` y métricas de CPU.
- `src/components/GlobalMetricsSummary.tsx`: Nuevo card visual de HB Shield.

## Instrucciones de Despliegue
1. Actualizar el código fuente.
2. **IMPORTANTE**: Borrar o comentar la línea `AUTH_URL` en `.env.local` para activar el modo multi-host.
3. Ejecutar `powershell -ExecutionPolicy Bypass -File prepar-despliegue.ps1`.
4. Ejecutar `node server.js` desde `dist-bizguard/`.
