# Implementación de Base de Datos NoSQL Embebida (V1.4)

## Fecha: 2026-03-20
## Objetivo
Sustituir el sistema de archivos JSON estático (`connectors.json`) por un motor de base de datos NoSQL embebido que permita persistencia atómica, consultas complejas y escalabilidad local en entornos Windows.

## Detalles Técnicos
- **Motor**: `@seald-io/nedb` (Fork moderno y mantenido de NeDB).
- **Arquitectura**: Documental NoSQL (local-first).
- **Archivos de Datos**:
  - `src/data/connectors.db`: Base de datos de conectores.
  - `src/data/metrics.db`: Almacén para métricas de tráfico y latencia.

## Proceso de Migración
Se ha implementado una rutina de migración automática (`migrateIfNeeded`) en `src/lib/connectors.ts` que:
1. Detecta si la base de datos está vacía.
2. Lee el archivo `connectors.json` si existe.
3. Importa todos los conectores a la nueva DB de forma atómica.
4. Mantiene la compatibilidad con las APIs existentes mediante casting de tipos.

## Beneficios
- **Integridad**: Evita corrupción de datos en cierres inesperados de Windows.
- **Rendimiento**: Búsquedas indexadas y lectura/escritura asíncrona optimizada.
- **Escalabilidad**: Preparado para almacenar histórico de métricas de varios días.
